# Editions And Licensing

RedShield Architect is planned as an MIT-licensed open-core product.

## Open Core

The open/core product should provide real local architecture value, including:

- local-first desktop/workbench experience
- requirements and practical UML modeling
- deterministic Git-friendly model package
- diagram views generated from semantic model data
- traceability between requirements, model elements, tests, decisions, and implementation tasks
- validation and CLI/pre-commit checks
- imports and exports for common architecture/modeling formats as they are implemented
- local AI proposal package mechanics with explicit provider configuration and reviewable typed operations
- local portfolio and enterprise-architecture objects that are useful within a project repository

## Paid Or Commercial Candidates

Paid/commercial scope is reserved for organizational scale, managed collaboration, governance, enterprise integration, and support-oriented capabilities, such as:

- hosted SaaS or managed self-hosted collaboration
- organization-wide inventory sync and estate discovery
- SSO, RBAC, tenant administration, audit administration, and enterprise policy management
- advanced integrations with CMDB/ITSM/cloud/security/compliance systems
- surveys, workflow automation, approvals, evidence packs, and enterprise reporting
- centralized AI policy controls, governed provider routing, compliance evidence, and managed review gates
- commercial support, training, packaging, migration tooling, and managed deployment

## Boundary Principle

The open core should not be a teaser shell. Local architects and technical teams should get a useful modeling, requirements, traceability, validation, and proposal-review workflow without paying for hosted or enterprise features.
